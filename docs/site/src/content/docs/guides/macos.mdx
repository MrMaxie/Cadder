## Direct use

From the directory that contains Cadder:

```sh
./caddy run
./cadder-tui
```

From any project directory where Cadder is on `PATH`:

```sh
caddy run
cadder-tui
```

## cadder.toml

Point Cadder at the real Caddy executable in [cadder.toml](../cadder-toml/). Use the path that matches your Caddy installation.

## Optional user PATH setup

If you want `caddy run` to go through Cadder everywhere, put Cadder's shim before real Caddy on your user `PATH`. Keep the real Caddy binary somewhere else and reference it from `cadder.toml`.

Homebrew Caddy is often at `/opt/homebrew/bin/caddy` on Apple Silicon and `/usr/local/bin/caddy` on Intel, but use `which caddy` to confirm your machine.

## TUI diagnostics

```sh
cadder-tui
```

Use the dashboard for daemon state, domains, logs, diagnostics, and activation toggles.

## Recover from common failures

- If Cadder cannot find real Caddy, check [cadder.toml](../cadder-toml/) first.
- If `caddy run` starts real Caddy instead of Cadder, your shell is finding real Caddy before the Cadder shim.
- If one project needs a different real Caddy binary, put a project-local `cadder.toml` in that project directory.
