---
title: TUI and diagnostics
description: Inspect daemon state, domains, per-domain logs, and diagnostics through cadder-tui.
---

`cadder-tui` opens the Cadder dashboard. It starts `cadderd` by default unless `--no-start` is passed.

```sh
cadder-tui
cadder-tui --no-start
cadder-tui --runtime-dir <runtime-dir>
```

## Views

- Overview shows runtime status, config status, entrypoint count, domain count, and active domain count.
- Entrypoints lists active shim registrations with source paths and domain counts.
- Domains groups domains with their source entrypoint and activation state.
- Logs opens a domain-specific log stream. Press `Enter` or `l` on a domain row to open that exact domain.
- Settings controls the minimum log severity shown in Logs.
- Diagnostics shows config and runtime diagnostics, including domain conflicts and runtime apply failures.

The TUI refreshes daemon state automatically on a short interval. Press `r` when you want an explicit refresh immediately.

## Controls

- `Tab`, `Shift+Tab`, `Left`, and `Right` switch views with wraparound.
- Arrow keys move selection.
- `Space` toggles the selected entrypoint or domain when the current view supports it.
- `/` enters search mode.
- In Settings, `Up` and `Down` choose `All`, `Info and higher`, `Warnings and errors`, or `Errors only`; `Enter` or `Space` applies the selected severity filter.
- `p` pauses or resumes log tailing in Logs.
- `Enter` manually refreshes the open Logs stream.
- `x` exports visible logs from the current stream.
- `d` asks the daemon to shut down.
- `q` exits the TUI.
