## What you need

- A real Caddy binary installed somewhere on your machine.
- A Cadder release for your OS from [GitHub Releases](https://github.com/MrMaxie/Cadder/releases/latest).
- A project with a `Caddyfile`.

## Configure real Caddy

Create `cadder.toml` next to the Cadder binaries and set the real Caddy command there. See [cadder.toml](../../user-guide/cadder-toml/) for Windows path syntax, project overrides, environment aliases, and the full set of supported command forms.

## Choose how commands are found

The cleanest setup is to keep real Caddy out of the `PATH` slot used for day-to-day `caddy run`, point Cadder at real Caddy in `cadder.toml`, and expose Cadder's shim as the `caddy` command.

You can also run Cadder by direct path. That is useful for testing or for setups where you do not want to change shell lookup order.

## Run a project

```sh
caddy run
```

The shim starts or connects to `cadderd`, registers the project Caddyfile, and keeps that registration alive while the command is running.

```sh
cadder-tui
```

Use the TUI to inspect current state, domains, logs, diagnostics, and toggles.
